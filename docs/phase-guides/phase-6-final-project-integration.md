# Phase 6 — Final Project Integration

## 0. Why This Phase Exists

The end goal is a coherent optimization project, not just a sequence of isolated exercises.

## 1. Learning Targets

- choose a compact optimization target
- keep a reference path and an optimized path
- produce correctness, benchmarking, and tradeoff evidence

## 2. Big Picture

This phase consolidates the repo into a portfolio-quality mini optimization lab.

## 3. Core Concepts Broken Down

- optimization target selection
- tradeoff analysis
- fallback design
- engineering narrative

## 4. Syntax and Keywords

- focus shifts from syntax to system design and explanation

## 5. Mental Execution Model

Think in layers: baseline → kernel/module → integration → measurement → write-up.

## 6. Minimal Worked Example

Use the toy block and imagine one optimized subpath plus one fallback rule.

## 7. Common Bugs and Pitfalls

- building something too large to explain
- optimizing everything without evidence

## 8. Before You Code

Explain what success for the final project actually means.

## 9. Mission

Follow `missions/phase-6-final-project.md`.

## 10. Test and Benchmark Checklist

- keep correctness tests first
- define measurement scope clearly
- document tradeoffs explicitly

## 11. Project Integration

The final project should mainly evolve `integration/`, supported by lower layers.

## 12. After You Finish

You should be able to present the project as a real engineering artifact.
